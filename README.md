# 🃏 德州扑克 CLI

> 一个能在命令行里玩的德州扑克游戏，不用联网，不用下载大文件，随时启动

---

## 1. 这是什么？

简单来说，这就是**德州扑克的命令行版本**，你可以在任何你想要打开的终端里运行一把德州扑克游戏。
谁不想急头白脸的打开命令行终端美美打一把德扑呢?

-  支持 2-8 人对战
-  AI 对手（还可以接入 LLM，让 AI 更聪明或更蠢）
-  ASCII 字符画扑克牌
-  纯 TypeScript 架构，轻量，方便任意运行

---

## 2. 如何能玩？

### 方法一：直接运行（推荐）

如果你拿到的是打包好的版本，直接双击 `holdem.exe` 就行。

或者如果你是从源码运行：

```bash
# 1. 进入项目目录
cd holdem-cli

# 2. 安装依赖（第一次需要）
npm install

# 3. 编译
npm run build

# 4. 开玩！
npm start
```

### 环境要求

- Node.js >= 16
- 一个能输入命令的终端（CMD、PowerShell、Git Bash、甚至浏览器终端 都行）

## 3. 游戏规则

### 基本流程

德州扑克的流程是固定的，每一轮都按这个顺序来：

| 阶段 | 发生了什么 | 你要做什么 |
|------|-----------|-----------|
| **翻牌前** | 每人发 2 张底牌 | 看牌，决定是否跟注、加注或弃牌 |
| **翻牌圈** | 发 3 张公共牌 | 结合底牌看牌力，继续下注 |
| **转牌圈** | 发第 4 张公共牌 | 牌局逐渐明朗，加注要谨慎 |
| **河牌圈** | 发第 5 张公共牌 | 最后一轮下注，决定是跟还是弃 |
| **摊牌** | 比大小 | 剩下的玩家亮牌，最大的拿走底池 |

### 你能做什么?

轮到你的时候，输入数字选择动作：

| 数字 | 动作 | 说明 |
|:----:|------|------|
| 1 | **弃牌** | 这局我不玩了，牌扔给荷官 |
| 2 | **过牌** | 不下注，先看看别人怎么玩 |
| 3 | **跟注** | 别人下多少，我跟多少 |
| 4 | **加注** | 我觉得牌不错，多下点 |
| 5 | **全押** | 梭哈！全部筹码压上 |

### 牌型大小（从大到小）

1. **皇家同花顺** - A-K-Q-J-10，同花色，天选之牌
2. **同花顺** - 五张同花色连号
3. **四条** - 四张一样的
4. **葫芦** - 三条 + 一对
5. **同花** - 五张同花色
6. **顺子** - 五张连号
7. **三条** - 三张一样的
8. **两对** - 顾名思义
9. **一对** - 两张一样的
10. **高牌** - 啥都不是，比谁的单张大

## 4. 自定义操作

### 接入 LLM 让 AI 更智能

游戏支持接入 OpenAI 兼容的 API（比如 OpenAI、DeepSeek、本地模型等），让 AI 对手用上大模型的大脑。

**怎么设置？**

启动游戏时，会询问是否管理 LLM 预设，选择 **是**，然后：

1. **新增预设** - 配置 API 地址、Key、模型名
2. **覆盖预设** - 修改已有的配置
3. **删除预设** - 删掉不用的

配置好后，开局时可以为每个 AI 玩家指定用哪个预设，或者直接使用普通 AI。

### 预设配置示例

```
预设名称: my-gpt
API Base URL: https://api.openai.com/v1
API Key: sk-xxxxxxxx
模型: gpt-4o-mini
temperature: 0.2
max tokens: 120
```

### 调试模式

想看游戏运行的详细日志？用调试模式启动：

```bash
npm run dev
# 或者
node dist/index.js --debug
```

日志会保存在 `.holdem-local/logs/` 目录下，方便排查问题。

### 打包成可执行文件

想分享给没装 Node.js 的朋友？打包成 exe：

```bash
npm run package
```

打包好的文件会在 `holdem/` 目录下。
